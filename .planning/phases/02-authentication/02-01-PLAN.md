  <action>
Add safe key handling:
- Generate API keys as high-entropy random tokens with a clear prefix (e.g., `msl_...`) to reduce copy/paste mistakes.
- Store only a non-reversible hash using stdlib `hashlib.pbkdf2_hmac` (avoid external hashing deps):
  - Derive: `pbkdf2_hmac('sha256', api_key_bytes, salt, iterations)`.
  - Use a per-key random salt (e.g., 16+ bytes via `secrets.token_bytes`).
  - Encode as an ASCII string that includes algorithm + iterations + salt + digest (e.g., `pbkdf2_sha256$210000$<salt_b64>$<digest_b64>`).
- Verify in constant time using `hmac.compare_digest`; never log full API keys.
- Tests: hashing round-trip, wrong key fails, output format includes prefix.
  </action>
  <verify>uv run pytest -q tests/test_api_key_hashing.py</verify>
  <done>Hashing + verification are tested, use stdlib PBKDF2, and never store plaintext secrets.</done>
